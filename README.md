# 📖 AI-DLC Foundations: Introduction

## ✨ Key Points

| Topic | Details |
|---|---|
| **AI-Assisted** | Narrow tasks like code completion with existing rituals slowing velocity |
| **AI-Managed** | Single-shot autonomous development with minimal human involvement, often requiring significant time fixing code issues |
| **Limitations** | AI-Assisted and AI-Managed approaches both have limitations in delivering transformative benefits |
| **AI-DLC Goal** | Balances AI’s heavy lifting with human oversight and control |
| **Phases** | Inception • Construction • Operations |
| **Workflow** | `Task → Plan → Clarify → Refine → Approve → Execute → Verify` |
| **Collaboration** | Cross-functional collaboration is essential for rapid decision-making |

# 📖 AI-DLC Foundations: Evolution

## ✨ Key Points

| Topic | Details |
|---|---|
| **Waterfall** | Sequential and phase-gated, appropriate for its era of expensive changes |
| **Agile** | Emerged when technical capabilities enabled iterative development |
| **AI Era** | Today’s AI capabilities represent another inflection point requiring fundamental methodology changes |
| **Challenge** | Retrofitting AI into existing processes creates mismatches between AI speed and human-paced ceremonies |
| **AI-DLC Vision** | Reimagines development with AI as integral, enabling continuous flow and real-time collaboration |

# 📖 AI-DLC Foundations: Principles

## 🌟 Nine Core Principles

| # | Principle |
|---|---|
| 1️⃣ | Reimagine workflows for the AI era rather than retrofitting AI into existing methodologies |
| 2️⃣ | AI orchestrates development while humans maintain ultimate control through validation and decisions |
| 3️⃣ | Design techniques are integral, not optional, building richer context for AI |
| 4️⃣ | Balance AI capabilities with human expertise through pragmatic alignment |
| 5️⃣ | Preserve proven practices while optimizing for AI-speed collaboration |
| 6️⃣ | Streamline roles to minimize handoffs while maintaining necessary oversight |
| 7️⃣ | Simplify to three phases (Inception, Construction, Operations) with strategic validation points |
| 8️⃣ | Enable flexible, AI-recommended workflows rather than rigid templates |
| 9️⃣ | Build on familiar concepts to ease practitioner transition |

# 📖 AI-DLC Workflow and Collaboration

## ✨ Key Points

| Topic | Details |
|---|---|
| **Workflow** | `Task → Plan → Clarify → Refine → Approve → Execute → Verify` — continuous cycle |
| **Decision Making** | Break problems apart, share responsibility, disagree & commit, decide with 70% of data |
| **Decision Types** | One-way door decisions require deliberation; two-way door decisions are reversible |
| **Critical Points** | Clarification questions, plan approval, execution intervention, artifact quality, iteration vs. completion |
| **Human Collaboration** | Human collaboration at AI speed prevents bottlenecks in decision-making |

# 📖 AI-DLC Foundations: Best Practices

## ✨ Key Points

| Topic | Details |
|---|---|
| **Task Management** | Never single-shot multi-step problems — break into manageable tasks |
| **Semantic Optimization** | Maximize semantics per token with meaningful info; optimize signal-to-noise ratio |
| **Context Management** | Refresh context strategically via documented decisions and relevant context loading |
| **AI Boundaries** | Set clear limits to prevent AI overreach; question unnecessary features |
| **Design Validation** | Ensure AI can implement patterns; pivot to simpler approaches when necessary |
| **Legacy Systems** | Contextualize legacy systems with semantic models before modernizing |
| **Refactoring** | Use surgical A/B comparison and cross-functional validation to preserve business logic |

# 📖 AI-DLC Foundations: Inception Deep Dive

## ✨ Key Points

| Topic | Details |
|---|---|
| **Purpose** | Answer Why (Intent), What (User Stories), How (Units of Work) |
| **Mob Elaboration** | AI proposes breakdowns; humans provide real-time validation across product, dev, QA |
| **Intent** | Expressed in business language; describes capabilities; avoids premature tech decisions |
| **User Stories** | Smallest negotiable slice delivering value; clear, testable acceptance criteria |
| **Units of Work** | Self-contained clusters of User Stories for parallel execution |
| **Success** | Teams can begin building immediately without further clarification meetings |

# 📖 AI-DLC Foundations: Construction Deep Dive

## ✨ Key Points

| Topic | Details |
|---|---|
| **Purpose** | Transform Units of Work into Deployment Units via Bolts (hours or days) |
| **Mob Construction** | Full team works collaboratively on shared screen; AI proposes, humans validate |
| **Stages** | Functional Design, NFR Design, Infrastructure Design, Code Generation, Build & Test |
| **Dynamic Staging** | AI may skip stages based on task complexity |
| **Functional Design** | Models business entities, rules, workflows in tech-agnostic language |
| **NFR Design** | Defines quality attributes: performance, scalability, security |
| **Infrastructure Design** | Maps components to cloud services; documented in ADRs |
| **Code Generation** | Begins after human approval; full traceability to requirements |
| **Build & Test** | Validates system via unit → integration → end-to-end testing with Mob Testing |
| **Conclusion** | Summary Report documents go/no-go decision for Operations Phase |

# 📖 AI-DLC Foundations: Operations Deep Dive

## ✨ Key Points

| Topic | Details |
|---|---|
| **Extension** | Operations applies AI-DLC collaboration principles to deployment, monitoring, incidents |
| **Deployment** | AI packages Units of Work; humans approve rollout plan |
| **Observability** | AI-DLC applied to monitoring: metrics, dashboards, alerting; humans ensure compliance |
| **Incident Management** | AI analyzes logs, identifies patterns, recommends actions; humans make final decisions |
| **Code Fixes** | Seamless transition from operations to development for fixes using AI-DLC workflow |
| **Transformation** | Turns operations from reactive to proactive, consistent system management |
